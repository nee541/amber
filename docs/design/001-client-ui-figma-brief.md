# 001 Amber 客户端界面 Figma 设计简报

## 1. 产品概述

Amber 是一个本地优先的物品档案管理客户端。当前 MVP 聚焦“商品生产日期、保质期存储与应用内提醒”，帮助用户手动记录商品，自动计算到期日期，并优先处理临期、今日到期和已过期的物品。

它不是电商、库存系统或营销型产品。界面应当像一个长期使用的本地管理工具：安静、清晰、可信、信息密度合理，打开后能快速知道“哪些东西快过期、哪些已经过期、接下来应该处理什么”。

## 2. 设计目标

- 让用户在首屏快速看到总体风险：活跃商品数、已过期数、临期数。
- 让商品列表具备高扫描效率：名称、分类、位置、到期日期、剩余天数、状态都应容易横向比较。
- 让新增商品流程保持轻量：必填字段少，可选字段不干扰主路径。
- 让提醒状态足够明确：临期和已过期应突出，但不要制造紧张感。
- 让设置页可理解：全局提醒、分类管理、位置管理、回收站入口应分组清楚。
- 移动端必须可用：列表改为卡片，详情和表单优先展示在列表上方或独立区域。

## 3. 用户与场景

主要用户是希望长期整理家庭物品、食品、药品、化妆品、家用品或收藏品的人。典型使用频率不是每天高强度操作，而是购买后录入、整理时查看、临期时处理。

关键场景：

- 购买食品后，快速录入名称、分类、生产日期、保质期。
- 查看药品、化妆品或食品中哪些即将过期。
- 将已处理的商品标记为已用完、已丢弃或已归档。
- 管理常用分类和存放位置，例如食品、药品、冰箱、药箱。
- 找回误删商品，或在确认后永久删除。

## 4. 信息架构

客户端使用单窗口工具型布局，主导航包含四个一级入口：

- 商品：默认入口，展示商品列表、筛选、详情和新增/编辑表单。
- 提醒：聚合已过期和即将过期商品，并提供快速处理动作。
- 设置：管理全局提醒规则、分类、存放位置和回收站入口。
- 回收站：查看已删除商品，支持恢复和永久删除。

桌面端使用左侧导航 + 右侧主内容的工具型布局。顶部标题区随当前页面变化，商品页额外显示搜索和新增商品操作。

商品页专属顶部区域包含：

- 搜索商品、分类或位置。
- 主操作按钮：新增商品。
- 三个摘要指标：活跃商品、已过期、临期。

提醒、设置、回收站页面不重复展示商品搜索和商品统计，避免跨页面信息语义混乱。

## 5. 核心页面设计

### 5.1 商品页

桌面端建议使用 Bento 卡片化左右分栏：

- 左侧为列表区，占主要宽度。
- 右侧为详情区。
- 当未选择商品时，右侧展示空状态。
- 新增和编辑商品使用居中弹窗，不占用右侧详情区域。

列表区内容：

- 标题：商品列表。
- 说明：默认优先显示已过期和临期商品。
- 数量标签：当前筛选结果数量。
- 工具栏：搜索、分类、位置、记录状态、到期状态。
- 表格列：名称、分类、到期日期、状态、数量、位置、操作。

列表排序表达：

- 已过期和临期优先。
- 同优先级内按到期日期从近到远。
- 已用完、已丢弃、已归档默认不出现，需通过状态筛选查看。

移动端建议将表格替换为商品卡片：

- 第一行：名称 + 状态标签。
- 第二行：到期日期 + 剩余/过期天数。
- 元信息：分类、位置、数量、用户状态。
- 操作：编辑、删除。

### 5.2 商品详情

详情区需要帮助用户判断这条记录的完整信息和下一步操作。

内容结构：

- 顶部：分类、商品名称、系统状态标签。
- 信息网格：生产日期、保质期、到期日期、剩余/过期、数量、存放位置、提醒设置、用户状态。
- 备注：仅在有内容时展示。
- 用户状态切换：在用/持有、已用完、已丢弃、已归档。
- 操作：编辑、删除到回收站。

设计重点：

- 到期日期和剩余/过期天数要比普通元数据更容易被看到。
- 状态切换应像一组分段按钮，而不是危险操作。
- 删除到回收站应使用危险色，但视觉权重低于编辑和状态处理。

### 5.3 新增/编辑商品表单

表单目标是快速完成单个商品录入。

交互形式：

- 新增和编辑均在居中弹窗中完成。
- 弹窗周围使用轻量虚化遮罩。
- 表单底部固定操作区，包含取消和保存。
- 这个模式也用于高风险确认操作，保持操作模型一致。

必填字段：

- 名称。
- 分类。
- 生产日期。
- 保质期数值。
- 保质期单位：天、月、年。

可选字段：

- 数量。
- 存放位置。
- 备注。
- 商品级自定义提醒提前天数。

表单顶部应有“到期预览”模块：

- 未输入完整日期信息时，提示填写生产日期和保质期后会计算到期日期。
- 输入有效后，展示到期日期、剩余/过期天数、系统状态、提醒来源。
- 当自定义提醒关闭时，说明使用全局默认提醒。

分类和位置交互：

- 支持输入已有选项，也支持输入新名称。
- 输入新分类或新位置后，保存时自动创建。

### 5.4 提醒页

提醒页用于集中处理需要注意的商品。

页面分为两组：

- 已过期。
- 即将过期。

每条提醒记录包含：

- 商品名称。
- 分类和位置。
- 到期日期。
- 剩余/过期天数。
- 状态标签。
- 快速动作：已用完、已丢弃、已归档。

设计重点：

- 已过期优先于即将过期。
- 两组都需要空状态。
- 快速动作应易点但不过度抢占注意力。

### 5.5 设置页

设置页是管理型页面，不应做成营销或说明页。

布局：

- 左列：分类管理。
- 右列：全局提醒、存放位置、回收站入口。

分区：

- 全局提醒：快捷值“当天、7 天、30 天、90 天”和自定义数字输入。
- 分类：新增、重命名、删除、迁移并删除。
- 存放位置：新增、重命名、删除、迁移并删除。
- 回收站入口：展示已删除商品数量并进入回收站。

分类和位置管理规则：

- 未被商品使用的项可以直接删除。
- 已被使用的项需要先选择迁移目标，再迁移并删除。
- 列表项内只展示名称、使用数量和删除入口。
- 删除与迁移并删除属于同一删除流程，应在居中弹窗中完成。
- 弹窗中根据使用数量决定显示“直接删除”或“选择迁移目标后删除”。
- 重命名保持在列表项内，删除逻辑不常驻占用分类列表空间。

### 5.6 回收站

回收站展示已删除商品。

每条记录包含：

- 商品名称。
- 分类、位置、到期日期。
- 删除时间。
- 操作：恢复、永久删除。

永久删除需要二次确认。Figma 设计中需要包含确认弹窗或确认态。

## 6. 关键组件

需要在 Figma 中建立可复用组件：

- 顶部标题区。
- 摘要指标 pill。
- 主导航 tabs。
- 状态标签：正常、临期、今日到期、已过期。
- 商品表格行。
- 移动端商品卡片。
- 筛选工具栏。
- 详情信息网格。
- 到期预览面板。
- 表单输入组。
- 分段状态按钮。
- 提醒记录行。
- 设置管理行。
- 空状态。
- 成功/错误 banner。
- 删除确认弹窗。

## 7. 状态与语义

系统计算状态：

- 正常：未进入提醒窗口。
- 临期：距离到期日期小于等于提醒提前天数。
- 今日到期：属于临期，但文案应单独显示。
- 已过期：当前日期晚于到期日期。

用户操作状态：

- 在用/持有。
- 已用完。
- 已丢弃。
- 已归档。

回收站状态：

- 未删除。
- 已删除。

建议颜色语义：

- 正常：低强调蓝色或中性色。
- 临期：琥珀/黄色。
- 今日到期：可沿用临期色，但文案更明确。
- 已过期：红色。
- 成功：绿色。
- 危险操作：红色。
- 普通背景：浅灰白。

## 8. 视觉方向

整体风格采用 Soft Bento Productivity：

- 浅色、低饱和、桌面效率工具质感。
- 背景使用暖灰 `#F7F7F5`，卡片使用白色表面。
- 大卡片圆角约 20px，按钮和输入框圆角约 12px。
- 使用清晰边框、柔和阴影和少量蓝紫高亮色。
- 不使用花哨 hero、插画背景、大面积渐变或装饰性图形。
- 按钮、输入框、选择器、标签、表格行保持统一高度和间距。

建议设计 token：

- 页面背景：接近 `#F8FAFC`。
- 卡片/面板背景：白色。
- 文字主色：深 slate。
- 次级文字：灰蓝。
- 主色：偏绿色，用于主按钮和焦点。
- 警告色：琥珀色。
- 危险色：红色。
- 基础圆角：8px。
- 输入高度：38px。
- 小按钮高度：34px。
- 主按钮高度：36px 以上。

## 9. 响应式断点建议

Figma 至少设计以下画板：

- Desktop：1440 x 900。
- Tablet：1024 x 768。
- Mobile：390 x 844。

布局变化：

- Desktop：商品页使用列表 + 详情双栏。
- 980px 以下：主内容改为单列，详情或表单置于列表上方。
- 720px 以下：表格改为商品卡片，顶部摘要换行。
- 560px 以下：导航从四列变为两列，表单字段单列排列。

## 11. 文案规范

文案应短、直接、偏工具语气。

推荐固定文案：

- 主标题：商品保质期。
- 说明：记录生产日期和保质期，优先处理临期与已过期商品。
- 主按钮：新增商品。
- 列表说明：默认优先显示已过期和临期商品。
- 空商品列表：新增商品后会出现在这里。
- 空详情：从列表打开详情，或新增一条商品记录。
- 空提醒：暂无记录。
- 回收站空状态：删除的商品会在这里恢复或永久删除。

避免：

- 营销式标语。
- 大段功能介绍。
- 在界面中解释太多实现细节。

## 12. 设计边界

当前 MVP 不应在界面中设计以下功能入口：

- 扫码识别。
- OCR。
- 图片、票据、PDF 归档。
- 批量导入。
- 云同步和账号。
- 系统通知。
- 自动联网补全商品资料。

可以在 Figma 的设计备注中标注这些是后续 feature，但不应进入当前客户端主界面。
